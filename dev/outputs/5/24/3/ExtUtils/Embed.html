ExtUtils::Embed provides utility functions for embedding a Perl interpreter and extensions in your C/C++ applications. Typically, an application Makefile will invoke http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"> http://www.w3.org/1999/xhtml"> perltidy

  • xsinit()

    Generate C/C++ code for the XS initializer function.

    When invoked as http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"> http://www.w3.org/1999/xhtml"> perltidy

    perltidy

    `perl -MExtUtils::Embed -e xsinit --` the following options are recognized:

    -o <output filename> (Defaults to perlxsi.c)

    -o STDOUT will print to STDOUT.

    -std (Write code for extensions that are linked with the current Perl.)

    Any additional arguments are expected to be names of modules to generate code for.

    When invoked with parameters the following are accepted and optional:

    http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"> http://www.w3.org/1999/xhtml"> perltidy

    perltidy

    xsinit($filename,$std,[@modules])

    Where,

    $filename is equivalent to the -o option.

    $std is boolean, equivalent to the -std option.

    [@modules] is an array ref, same as additional arguments mentioned above.

  • ldopts()

    Output arguments for linking the Perl library and extensions to your application.

    When invoked as http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"> http://www.w3.org/1999/xhtml"> perltidy

    perltidy

    `perl -MExtUtils::Embed -e ldopts --` the following options are recognized:

    -std

    Output arguments for linking the Perl library and any extensions linked with the current Perl.

    -I <path1:path2>

    Search path for ModuleName.a archives. Default path is http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"> http://www.w3.org/1999/xhtml"> perltidy

    perltidy

    @INC . Library archives are expected to be found as /some/path/auto/ModuleName/ModuleName.a For example, when looking for Socket.a relative to a search path, we should find auto/Socket/Socket.a

    When looking for http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"> http://www.w3.org/1999/xhtml"> perltidy

    perltidy

    DBD::Oracle relative to a search path, we should find auto/DBD/Oracle/Oracle.a

    Keep in mind that you can always supply /my/own/path/ModuleName.a as an additional linker argument.

    -- <list of linker args>

    Additional linker arguments to be considered.

    Any additional arguments found before the -- token are expected to be names of modules to generate code for.

    When invoked with parameters the following are accepted and optional:

    http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"> http://www.w3.org/1999/xhtml"> perltidy

    perltidy

    ldopts($std,[@modules],[@link_args],$path)

    Where:

    $std is boolean, equivalent to the -std option.

    [@modules] is equivalent to additional arguments found before the -- token.

    [@link_args] is equivalent to arguments found after the -- token.

    $path is equivalent to the -I option.

    In addition, when ldopts is called with parameters, it will return the argument string rather than print it to STDOUT.

  • Examples http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"> http://www.w3.org/1999/xhtml"> perltidy

    perltidy

     perl -MExtUtils::Embed -e ldopts

    perltidy

    libperl and extensions found in http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"> http://www.w3.org/1999/xhtml"> perltidy

    perltidy

    $Config{libs} and the first ModuleName.a library for each extension that is found by searching http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"> http://www.w3.org/1999/xhtml"> perltidy

    perltidy

    @INC or the path specified by the -I option. In addition, when ModuleName.a is found, additional linker arguments are picked up from the extralibs.ld file in the same directory.

    http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"> http://www.w3.org/1999/xhtml"> perltidy

    This will do the same as the above example, along with printing additional arguments for linking with the http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"> http://www.w3.org/1999/xhtml"> perltidy

    Socket extension.

    http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"> http://www.w3.org/1999/xhtml"> perltidy
