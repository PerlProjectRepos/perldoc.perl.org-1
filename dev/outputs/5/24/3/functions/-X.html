    A file test, where X is one of the letters listed below. This unary operator takes one argument, either a filename, a filehandle, or a dirhandle, and tests the associated file to see if something is true about it. If the argument is omitted, tests http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd> http://www.w3.org/1999/xhtml> perltidy perltidy $_ , except for http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"> http://www.w3.org/1999/xhtml"> perltidy

    perltidy

    1 for true and http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"> http://www.w3.org/1999/xhtml"> perltidy

    perltidy

    '' for false. If the file doesn't exist or can't be examined, it returns undef and sets http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd> http://www.w3.org/1999/xhtml> perltidy perltidy $! (errno). Despite the funny names, precedence is the same as any other named unary operator. The operator may be any of:

    1.     -r  File is readable by effective uid/gid.
    2.     -w  File is writable by effective uid/gid.
    3.     -x  File is executable by effective uid/gid.
    4.     -o  File is owned by effective uid.
    6.     -R  File is readable by real uid/gid.
    7.     -W  File is writable by real uid/gid.
    8.     -X  File is executable by real uid/gid.
    9.     -O  File is owned by real uid.
    11.     -e  File exists.
    12.     -z  File has zero size (is empty).
    13.     -s  File has nonzero size (returns size in bytes).
    15.     -f  File is a plain file.
    16.     -d  File is a directory.
    17.     -l  File is a symbolic link (false if symlinks aren't
    18.         supported by the file system).
    19.     -p  File is a named pipe (FIFO), or Filehandle is a pipe.
    20.     -S  File is a socket.
    21.     -b  File is a block special file.
    22.     -c  File is a character special file.
    23.     -t  Filehandle is opened to a tty.
    25.     -u  File has setuid bit set.
    26.     -g  File has setgid bit set.
    27.     -k  File has sticky bit set.
    29.     -T  File is an ASCII or UTF-8 text file (heuristic guess).
    30.     -B  File is a "binary" file (opposite of -T).
    32.     -M  Script start time minus file modification time, in days.
    33.     -A  Same for access time.
    34.     -C  Same for inode change time (Unix, may differ for other
    35. 	platforms)

    These operators are exempt from the "looks like a function rule" described above. That is, an opening parenthesis after the operator does not affect how much of the following code constitutes the argument. Put the opening parentheses before the operator to separate it from code that follows (this applies only to operators with higher precedence than unary operators, of course):

    1.     -s($file) + 1024   # probably wrong; same as -s($file + 1024)
    2.     (-s $file) + 1024  # correct
